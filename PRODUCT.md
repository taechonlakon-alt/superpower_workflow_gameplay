# Product

## Register

brand

## Users

Developers, AI builders, and learners who want to understand structured AI-assisted workflow through a short playable browser game. They are not trying to operate a productivity dashboard. They are playing through a tiny project survival scenario where each choice reveals how planning, review, tools, and AI usage affect delivery risk.

## Product Purpose

SUPERPOWER WORKFLOW is a single-screen browser game that teaches why workflow matters when using AI. The player guides an AI-assisted landing page project through brainstorm, plan, execute, review, chaos events, emergency recovery, and a final report. Success means the interface feels immediately like a playable pixel craft adventure while preserving the exact gameplay logic and visible Resource Bar model.

## Brand Personality

Playful. Survival-craft. Sharp.

The game should feel bright, readable, tense when the project is in danger, and memorable enough that players recognize their workflow pattern after one run.

## Anti-references

This must not look like a dashboard, admin panel, SaaS app, terminal UI, developer OS, monitoring system, cyberpunk grid, tactical command center, analytics report, or productivity tool. Dark app-shell layouts, sidebars, system-monitor cards, tiny technical labels, and mostly black or dark-brown screens are explicit failures.

It must also avoid copying Minecraft or Terraria assets, logos, sprites, blocks, mobs, textures, or exact UI. The target is an original pixel craft adventure mood, not a clone.

## Design Principles

1. Read as a game first. Every screen should feel like a pixel craft adventure interface before it reads as a web app.
2. Preserve the workflow lesson. Visual polish must keep Time, Token, and Risk visible during play without making risky choices look like cartoonishly wrong answers.
3. Use world materials as interface language. Sky, grass, dirt, wood, stone, parchment, inventory slots, quest boards, survival meters, and mission boards should carry the UI.
4. Make pressure visible without dashboards. The Resource Bar should feel like survival HUD meters, chaos like danger encounters, and reports like mission debriefs.
5. Keep Thai text readable. Pixel flavor supports the interface, but legibility wins over decorative density.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Thai and English text must remain readable on mobile and desktop. Interactive targets should be large enough for touch. Keyboard focus must remain visible. Motion should respect reduced-motion preferences.
